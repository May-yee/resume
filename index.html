自我介紹

我是王媚儀，學生期間曾在飲料店工作並擔任畢籌會常務組組長，畢業後進入愛貝斯擔任前端工程師。在一年的工作中參與製作了17個網站專案，接觸各種產業後發現每個客戶對網站都有不同的需求與挑戰。透過與客戶、企劃、設計師的溝通協調來滿足各方需求的過程，讓我漸漸對專案管理產生興趣，也發現自己相較於寫程式，更喜歡需要規劃與溝通的專案管理工作。

在專案執行過程中，我展現了快速學習與負責任的態度，也培養出細心的工作習慣。當遇到技術問題時，我能用簡單易懂的方式向企劃說明技術困難點，有效溝通解決問題。加上服務業累積的溝通技巧，讓我在面對客戶時能靈活應對各種狀況。

平時我熱愛參觀各種展覽，從中觀察細節與學習創作靈感，也經常瀏覽 WebDesignClip 等設計網站，關注最新的網頁設計趨勢與創意表現手法，並且因自己喜歡製作動畫效果，所以也喜歡觀察其他網站的動畫效果。

INTRODUCTION
WEB PROJECT
2024 SEP.
AMICA

此專案為家電品牌，網站首頁需要多個區塊展示不同產品資料，面臨複雜的資料抓取與整合挑戰。同時，客戶要求在 Banner 區塊嵌入品牌影片，這些技術需求對當時的我來說是相當大的挑戰。

後台架構優化
客戶產品有兩個系列，各自包含不同類型家電。為避免重複上稿，我將欄位與單元整合設計。但經過多個專案經驗累積後，發現針對此類產品展示區塊獨立設計後台單元，反而能降低複雜度，因為產品類別變動性低，且不是所有類別都需在此區塊顯示，大幅提升客戶操作便利性與程式維護效率。
技術問題解決
影片採用 YouTube 套件時遇到 RWD 適應以及載入問題，無法隨意調整尺寸，且客戶不希望出現黑色留白。經過多次測試與調整，優化了影片載入機制，並成功讓影片能完整填滿螢幕，解決響應式設計的適應問題。
UX 優化提案
「關於我們」區塊的長圖在手機版占滿整個螢幕，但設計稿未設計切換箭頭。我主動向企劃提議在文字區塊加入箭頭指示，讓使用者清楚知道這是可切換的輪播功能，提升使用體驗。
VISIT WEBSITE
2024 OCT.
日健建設

此專案設計感十足，包含大量視覺裝飾元素，對前端技術實現具有高度挑戰性。

客戶需求溝通
客戶對 loading 動畫效果要求極為嚴格，經過多次來回討論與修改，最終完成符合客戶期望的動畫表現。
使用者體驗優化
設計稿中有大量裝飾性線條和房屋元素，初期計劃全部製作成 SVG 畫線動畫，但在製作過程中發現會干擾使用者的閱讀順序及增加網站載入效能。經評估後主動調整策略，僅保留適當的動畫效果，平衡視覺豐富度與使用者體驗。
技術難點突破
工程案例區塊採用圓形輪播切換設計，這是當時技術上的一大挑戰。我投入大量時間研究相關 CSS 技術，反覆測試與改寫程式碼，最終實現流暢的圓形輪播效果，並確保程式碼結構清晰易於後續維護。
